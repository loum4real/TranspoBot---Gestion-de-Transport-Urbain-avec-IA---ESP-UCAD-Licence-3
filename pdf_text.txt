  1. CONTEXTE ET OBJECTIFS
Dans le cadre du cours d'integration de l'IA dans les systemes d'information (Licence 3 GLSi), vous devez
concevoir  et  realiser  une  application  web  de  gestion  de  transport  urbain  integrant  un  assistant
conversationnel base sur un LLM (Large Language Model) connecte a une base MySQL.
L'objectif  est  de  demontrer  votre  capacite  a  concevoir  un  systeme  complet  :  modelisation,
developpement backend, integration LLM et deploiement en ligne.
  2. DESCRIPTION DU PROJET
Une societe de transport gere sa flotte, ses chauffeurs, ses trajets et ses incidents. L'application permet
aux gestionnaires d'interroger ces donnees en langage naturel via un chatbot IA.
  3. FONCTIONNALITES REQUISES
3.1  Gestion des donnees (interface web classique)
-  Afficher la liste des vehicules avec leur statut
-  Afficher la liste des chauffeurs
-  Afficher les trajets recents
-  Tableau de bord avec indicateurs cles (KPI)
3.2  Assistant IA conversationnel (obligatoire)
-  Repondre a des questions en langage naturel (francais ou anglais)
-  Generer automatiquement les requetes SQL (Text-to-SQL)
-  Afficher les resultats sous forme de tableau
-  Exemples : "Combien de trajets cette semaine ?", "Quel chauffeur a le plus d incidents ?"
-  Securite : uniquement des requetes SELECT (pas de modification)
  4. BASE DE DONNEES MYSQL
Schema fourni (fichier schema.sql). Tables minimales requises :
-  vehicules, chauffeurs, lignes, tarifs, trajets, incidents
-  Donnees de test incluses dans le script SQL
-  Vous pouvez enrichir le schema si necessaire
  5. ARCHITECTURE TECHNIQUE
-  Backend : Python FastAPI (ou Node.js Express)
-  Base de donnees : MySQL 8.x
-  LLM : OpenAI API (gpt-4o-mini recommande) ou Ollama (LLaMA3/Mistral en local)
-  Frontend : HTML/CSS/JS (ou React/Vue - optionnel)
-  Deploiement : Railway, Render, ou serveur VPS
-   Interface  de  chat  :  ClawdBot  self-hosted,  bot  Telegram,  bot  Discord,  ou  chat  web  integre  au
backend

  6. EXEMPLES DE DIALOGUES LLM ATTENDUS
Voici des exemples representatifs du niveau de precision attendu pour l assistant IA :
-  Utilisateur : "Combien de trajets ont ete effectues cette semaine ?"
   -> LLM genere : SELECT COUNT(*) FROM trajets WHERE date_heure_depart >= DATE_SUB(NOW(),
INTERVAL 7 DAY) AND statut='termine'
   -> Reponse : "12 trajets termines cette semaine."
-  Utilisateur : "Quel chauffeur a le plus d incidents ce mois-ci ?"
   -> LLM genere : SELECT c.nom, c.prenom, COUNT(i.id) as nb FROM incidents i JOIN trajets t ON
i.trajet_id=t.id  JOIN  chauffeurs  c  ON  t.chauffeur_id=c.id  WHERE
MONTH(i.date_incident)=MONTH(NOW()) GROUP BY c.id ORDER BY nb DESC LIMIT 1
   -> Reponse : "Ibrahima FALL avec 2 incidents ce mois."
-  Utilisateur : "Quels vehicules necessitent une maintenance ?" (question hors SQL direct)
   -> Reponse : "Voici les vehicules en statut maintenance : DK-9012-EF (78 000 km)."
